# Report Index

Reports are dated evidence artifacts. They are not automatically current architecture.

## Current pointers

- See `latest.md` for current report pointers by domain.

## Report classes

- Current signoff: validated against current code or runtime behavior.
- Current investigation: recent, still relevant, but may not be a signoff.
- Historical: useful for background, not default retrieval.
- Superseded: retained for audit; load only when investigating history.

## Retention rules

- Keep dated filenames.
- Add summary/frontmatter when promoting a report into default retrieval.
- Link reports back to code, migrations, evals, or decisions where possible.
- Do not treat old reports as source of truth when code or migrations disagree.

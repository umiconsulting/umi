## Adapters

- **tenant.ts** — `getTenantConfig(slug)`/`getTenant(slug)` → legacy `Tenant` shape (camelCase)
  - canonical ids `programId`/`businessId`/`subscriptionId` + `locations`; `getActivePromo(t)`;
    `requireActiveSubscription(t)`.
- **wallet.ts** — `creditWallet({tenantId, cardId, deltaCents, type:'topup'|'purchase'|'adjustment'|'gift_card_redeem', reason?, idempotencyKey, staffMemberId?, description?, sourceType?, sourceId?})` → `{balanceCents}`; `applyWalletDelta(tx, d)` to compose inside `prisma.$transaction`; `getWalletBalance(tenantId, cardId)`. **NEVER write a balance column directly.** Money is already centavos (no ×100); debit = negative `deltaCents`.
- **identity.ts** — `resolveContact({tenantId, kind:'phone'|'whatsapp'|'email', rawValue, displayName?})` → personId (find-or-create `core.people`+`contact_methods`, dedup by normalized phone); `normalizePhone`; `findPersonByPhone/Email`; `findLoginUser(tenantId, email)` → `{user, membership, roleKeys}`; `getStaffMemberId(tenantId, userId)`; `createSession/findSessionByToken/deleteSessionByToken/deleteExpiredSessions`.
- **auth.ts** — `createSession(subjectId, role, tenantId)` (unchanged API; CUSTOMER role→person_id, else user_id), `verifyAccessToken/RefreshToken`, `signQRPayload`, `signWalletBarcode`, `verifyQRPayload`, `hashPassword/verifyPassword`, `getAuthUser`, `requireAuth`, `generateRandomToken`, `generatePassSerial` — ALL unchanged; keep using.
- **prisma-helpers.ts** — `getActiveRewardConfig(tenantId)`, `rewardConfigDefaults(cfg)` → `{visitsRequired, rewardName, rewardDescription}`, `findCardByIdentifier(identifier, tenantId, {person?})` → card (+accounts +balances, +person when asked).

## Recipes

- **Account/card create:** programId = `getTenantConfig(slug).programId`; account = `prisma.accounts.findFirst/create({tenant_id, person_id, program_id})`; card = `prisma.cards.create({tenant_id, account_id, card_number, qr_token, status:'active'})`.
- **Staff attribution:** `staff_member_id = await getStaffMemberId(tenantId, auth.sub)`.
- **Money:** `creditWallet(...)` (idempotencyKey e.g. `topup_${cardId}_${Date.now()}`). For atomic multi-table (gift redeem) use `prisma.$transaction(tx => applyWalletDelta(tx, ...))`.
- **Login:** `findLoginUser(tenantId, email)`; `verifyPassword(pw, user.password_algorithm==='scrypt-sha256-v1' ? \`scrypt:${user.password_salt}:${user.password_hash}\` : \`${user.password_salt}:${user.password_hash}\`)`; role from roleKeys (owner/admin→ADMIN, staff/cashier→STAFF); `createSession(user.id, role, tenantId)`.
- **Gift redeem:** `$transaction` → set gift_cards.redeemed_at + redeemed_loyalty_card_id; gift_card_ledger −amount; `applyWalletDelta(tx, {type:'gift_card_redeem', deltaCents:+amount, idempotencyKey:\`giftredeem_${id}\`, sourceType:'gift_card', sourceId:id})`.
- **Tenant write fan-out:** target the owning table by its tenant_id (ids from getTenantConfig).
- **Passes:** look up the card's provider pass in `prisma.passes`; create if absent (NEW pass → generate serial/auth/object id; MIGRATED cards already have rows — never regenerate). PassKit register/unregister → upsert/delete `prisma.pass_devices`. Keep ALL secret/cert/JWT logic identical.

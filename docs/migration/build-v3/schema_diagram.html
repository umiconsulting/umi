read this first · the design thesis is the layout

Three schemas become three lanes. umi is sealed authority (Umi owns every fact; no café writes here), tenant is the café's durable business facts under row-level security, and runtime is sealed working memory the running code reads back to act next. The bands between lanes are the only cross-schema foreign keys: identity flows up into one umi.user; ownership flows down via business_id. Money & stamps live in tenant append-only ledgers (balance = SUM, never cached); embeddings & FSM state live in runtime, never in tenant.
